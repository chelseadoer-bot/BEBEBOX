1 서비스 개요 · 핵심 전략

아기 성장 기록(육아일기)과 AI 콘텐츠를 가족·지인과 공유하고, 그 흐름 안에서 자연스럽게 출산/육아 선물(레지스트리)을 주고받는 모바일 웹 서비스.

핵심 우회 전략 (Why)

한국 정서상 "내 선물 링크 보세요"는 민망하다. 그래서 육아일기 + AI 카드를 만들어 지인에게 공유하고, 지인은 "아기 보러" 자연스럽게 들어왔다가 레지스트리를 발견해 선물한다. 모든 활동은 캔디(🍬)로 보상되어 행동→보상→소비→교환 루프로 묶인다.

✍️ 일기 작성› ❤️ 지인 공감› 🎁 선물 도착(선반)› 🍬 캔디 적립› ✨ AI·게임·상품권

2 시스템 아키텍처

레이어구현비고
웹 서버server.py — Python 표준 라이브러리 http.server외부 의존성 없음(포트 8080, directory=ROOT 정적 서빙)
데이터SQLite data/bebebox.db (db.py)테이블: photos · family_data · sessions
이미지uploads/ 디렉터리배포 시 영구 디스크 /data 마운트
프론트엔드바닐라 JS SPA (index.html + js/*)빌드 없음, ?v=N 캐시버스팅
인증카카오 OAuth (kakao_auth.py + 세션 쿠키)공유는 Kakao JS SDK
AI 백엔드ai_backend.py + apps/*/backend/Gemini API, 앱별 파이프라인/스키마
외부 연동키디키디(이랜드몰) 상품 검색 프록시/api/kidikidi/search → elandmall
동기화가족코드 단위 family_data 전체 JSON기기 간 공유·재배포 보존
📌 데이터는 가족코드(초대코드) 하나로 묶인다. 같은 코드를 쓰는 기기끼리 일기·위시·선물·캔디가 모두 공유된다.

3 3개 화면(서피스)

📱
사용자index.html부모(회원) 앱. 일기·피드, 위시리스트/선반, 캔디, AI 게임, 설정. 카카오 로그인 + 온보딩.
🔗
공유share.html지인이 카톡 링크로 진입하는 페이지. 일기 먼저, 하단 플로팅으로 선물(위시·받은 선반)·방명록. 로그인 불필요.
🛠️
운영자admin.html관리자 키 게이트. 쿠폰 지급 대기열, 회원/상세, 지인 전환 펀넬, 게임 탭 배너 편집.

+ AI 미니앱 8종은 각각 apps/<slug>/index.html 로 사용자 앱 안에서 iframe으로 열린다.

4 핵심 기능 상세

① 온보딩 사용자

  • 카카오로 시작 → 부모 / 초대받음(지인·배우자) 분기
  • 역할(엄마·아빠·가족) → 아기 이름(태명/“아직 안 정했어요” 지원) → 키디키디 연동(선택) → 완료
  • 초대받음은 6자리 가족코드로 기존 가족 공간에 합류(게스트 모드)

② 일기 · 피드 사용자

  • 사진/글 기록(인스타식 컴포저), 월령 연령 탭, 좋아요·댓글
  • 댓글 시트(밝은 흰색 테마)
  • 성장 저니(스케치 정원) — 여정 노드·퀘스트, raid 펀딩(유모차 성문 등)

③ 위시리스트 · 받은 선물 선반 사용자 공유

  • 임신 시기/월령별 스테이지(s1~s8), 우선순위 필수·권장·선택, 대상 mom·baby
  • 위시 카드 탭 한 번 → 통합 시트(공유 위시 공개 · 받음 기록 · 인사말 · 상품 선택 · 삭제)
  • 받은 선물 선반: 선물받으면 준 사람과 함께 선반에 채워짐. 탭하면 준 사람이 남긴 인사말 표시(공유 페이지 “축하 한마디” guestbook 연동)
  • 달성률: “받고 싶은 선물 N/M개 도착” + “선물마다 +50🍬”
  • 키디키디 추천 상품(7초 타임아웃 + 폴백 링크로 무한로딩 방지)

④ 오늘의 미션 사용자

사진·공유·AI 활동으로 9칸을 채우면 +100🍬 보너스(하루 1회). 날짜가 바뀌면 새 판. ※ 선물 레지스트리의 ‘선물 퍼즐’과는 별개의 데일리 캔디 미션.

⑤ 공유 · 바이럴 공유

  • 카카오톡 공유: Kakao JS SDK Kakao.Share.sendDefault(피드 카드) → Web Share → 링크복사 폴백
  • 공유 링크 /share/<가족코드> 에 OG 메타(제목·이미지) 서버 주입 → 카톡 미리보기 카드
  • 지인이 선물 완료 시 관계·이름·축하 메시지를 방명록(guestbook)에 남김 → 부모 선반/운영자에 반영
  • “나도 우리 아이 기록 시작하기” 2차 바이럴 CTA

⑥ 운영자 대시보드 운영자

  • KPI: 회원 수 · 지급 대기 쿠폰 · 발급 완료 · 누적 기록 · 지인 선물 완료
  • 쿠폰 지급 대기열: 캔디 교환 완료 회원에게 키디키디 쿠폰 발급 체크(키디 아이디 없음 경고)
  • 회원 목록/검색 + 상세(위시·받은선물·선물해준 사람·관계·메시지)
  • 지인 유입 전환 펀넬: 조회 → 선물클릭 → 완료 / 하트·댓글·미니앱
  • 게임 탭 상단 배너 편집(다중·이미지·아이콘·링크, 기본 배너 항상 유지)
